# Specification: Security Hardening

**Track ID:** security-hardening_20260310
**Type:** Bug
**Created:** 2026-03-10
**Status:** Draft

## Summary

Remediate all known vulnerabilities in the NetRunner ecosystem by updating Go dependencies, the Go runtime, and Python requirements to their minimum secure versions.

## Context

A comprehensive security scan identified vulnerabilities across the backend (Go) and worker (Python) environments. This track ensures the entire appliance is hardened against known exploits including predictable UUIDs, memory exhaustion, and sensitive token exposure.

## Problem Description

The project is currently exposed to:
- **Critical (Go)**: Predictable UUID generation in `gofiber/utils@1.1.0`.
- **High (Go)**: Memory exhaustion during token parsing in `oauth2@v0.0.0...`.
- **High (Python)**: Spotify auth token exposure in `spotipy@2.23.0` (cache file permissions).
- **Multiple STDLIB (Go)**: DoS and quadratic complexity flaws in `stdlib@1.25.0`.
- **Low (Python)**: XSS in `spotipy` OAuth callback server.

## Acceptance Criteria

- [ ] `github.com/gofiber/utils` is updated to at least `v1.2.0`.
- [ ] `golang.org/x/oauth2` is updated to at least `v0.27.0`.

- [ ] `spotipy` in `ops/worker/requirements.txt` is updated to at least `2.25.2`.
- [ ] All services build and existing tests pass (both Go and Python).
- [ ] Vulnerability scan returns 0 findings for these packages.
