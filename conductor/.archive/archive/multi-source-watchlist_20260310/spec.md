# Specification: Multi-Source Watchlist Expansion

**Track ID:** multi-source-watchlist_20260310
**Type:** Feature/Refactor
**Created:** 2026-03-10
**Status:** Draft

## Summary

Pivot the NetRunner Watchlist engine from a Spotify-only integration to a universal, provider-agnostic system. Implement dedicated modules for Last.fm, ListenBrainz, RSS, Bandcamp, Discogs, and Local File imports (M3U, CSV, TXT).

## Context

The current `WatchlistService` is tightly coupled to the Spotify Web API, which now requires a Premium subscription for developer access. To maintain automated discovery, the system must support alternative, open sources for tracking "Loved" items, playlists, and new releases.

## User Story

As a music collector, I want to be able to integrate as many sources as possible for curated music acquisition so that I can maintain an automated discovery pipeline without relying on a single, restrictive streaming provider.

## Acceptance Criteria

- [ ] **Universal Watchlist Interface**: Refactor `WatchlistService` to use a provider-based architecture where each source (Last.fm, RSS, etc.) is a separate module.
- [ ] **Open Provider Support**: Implement modules for **Last.fm** (Loved Tracks/Top Tracks) and **ListenBrainz**.
- [ ] **Discovery Feed Support**: Implement modules for **RSS feeds** (Bandcamp, etc.) and **Discogs Wantlist**.
- [ ] **Local File Import**: Implement a module for processing **M3U, CSV, and TXT** files as a one-time or watched source.
- [ ] **Data Integrity**: Ensure the same deduplication and library-check logic applies consistently across all new sources.

## Dependencies

- **WatchlistService Refactor**: Removal of hardcoded Spotify logic.
- **Provider APIs**: Public/Free access to Last.fm and ListenBrainz APIs.

## Out of Scope

- Implementing paid or subscription-only developer APIs.
- Automated Spotify playback control.

## Technical Notes

- Use a `WatchlistProvider` interface in Go to abstract the `FetchTracks` logic.
- Store provider-specific configuration (API keys, URLs) in the `Watchlist.Config` JSONB field.
