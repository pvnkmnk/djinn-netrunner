## Summary

Migrate the `ops-web` Python FastAPI application to a high-performance Go Fiber implementation. This includes porting all REST endpoints, authentication logic, HTMX-compatible template rendering, and WebSocket log streaming.

## Context

The project is transitioning from a hybrid Python/Go architecture to a unified Go-based backend to improve performance, concurrency, and maintainability. The core worker has already been migrated; the Web API is the final major component remaining in Python.

## Acceptance Criteria

- [ ] Go API serves all routes previously handled by Python (`/`, `/api/*`, `/ws/*`, `/static/*`).
- [ ] Authentication remains seamless (existing users can log in, sessions are preserved).
- [ ] HTMX frontend functionality is fully preserved (dashboard stats, source management, modal interactions).
- [ ] WebSocket console correctly streams real-time job logs via PostgreSQL NOTIFY.
- [ ] Python `ops-web` container and associated files can be safely removed.

## Dependencies

- **Complete Track:** `go_transition_20260304` (Core Go Worker Setup)
- **Database:** PostgreSQL (existing schema)
- **Libraries:** Go Fiber, GORM, `lib/pq` (for NOTIFY), `golang.org/x/crypto/bcrypt`.

## Out of Scope

- Redesigning the UI/UX (visual parity is the goal).
- Implementing new features not present in the Python version.
- Migration of the Caddy reverse proxy logic (only the backend target changes).

## Technical Notes

- **Password Hashing:** Use `golang.org/x/crypto/bcrypt` to match Python's `passlib`.
- **Templates:** Use Fiber's `html` template engine. Implement custom functions to replicate Jinja2's `strftime`.
- **Concurrency:** Leverage Go's native concurrency for the WebSocket connection manager and NOTIFY listener.
