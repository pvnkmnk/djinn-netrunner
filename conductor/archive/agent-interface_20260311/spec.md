# Specification: Agentic Interface (MCP & CLI)

**Track ID:** agent-interface_20260311
**Type:** Feature
**Created:** 2026-03-11
**Status:** Draft

## Summary
Implement a high-fidelity "Agentic Surface" for NetRunner, consisting of a built-in **Model Context Protocol (MCP)** server and an **Agent-Native CLI**. This enables AI agents to autonomously bootstrap the system, manage watchlists, monitor downloads, and tune configurations.

## Context
NetRunner is currently managed via a web UI or direct database/config edits. By exposing its internals through standard agent protocols (MCP), users can delegate complex operations (e.g., "Fix all my broken downloads" or "Find and track the top 10 labels on Bandcamp") to their AI assistant.

## User Story
As an automated home lab operator, I want my AI assistant to be able to setup and operate NetRunner so that I can delegate library curation and system maintenance tasks autonomously.

## Acceptance Criteria
- [ ] **Embedded MCP Server**: A standard-compliant MCP server running within the NetRunner binary.
- [ ] **Operational Toolset**: Implementation of core tools: `get_system_status`, `list_watchlists`, `add_watchlist`, `get_job_logs`, and `restart_worker`.
- [ ] **JSON-Native CLI**: A CLI mode that outputs parsable JSON for all management commands.
- [ ] **Bootstrap Protocol**: A specific tool/command that allows an agent to validate the environment (Docker, paths, permissions) and perform initial configuration.
- [ ] **Security Scoping**: A dedicated `API_KEY` mechanism for the MCP server to ensure only authorized agents can control the appliance.

## Dependencies
- **Core Services**: Access to `WatchlistService`, `Database`, and `WorkerOrchestrator`.
- **MCP Go SDK**: `github.com/metoro-io/mcp-golang` or similar.

## Out of Scope
- Voice-based control (TTS/STT).
- Integration with non-MCP LLM protocols.

## Technical Notes
- Use `spf13/cobra` for the CLI to handle the `--json` flag and nested command structure.
- The MCP server should be a separate entry point in `backend/cmd/agent` but share the same core logic.
