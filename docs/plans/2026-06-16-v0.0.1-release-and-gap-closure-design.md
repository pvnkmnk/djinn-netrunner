## Overview

Single unified milestone to release NetRunner v0.0.1 while closing all known gaps from TODO.md and the pre-release checklist. Work is organized into 3 parallel implementation streams (Œ±, Œ≤, Œ≥) plus a final integration/release stream (Œ¥).

## Stream Œ±: Infrastructure

### yt-dlp + Node.js in Docker

- **Goal**: Fix `TestYtdlpService_DownloadAudio` by shipping a current yt-dlp and Node.js JS runtime in Docker images.
- **Changes**:
  - `ops/Dockerfile` (shared base or per-service Dockerfile): Install `nodejs` (22 LTS) and `yt-dlp` via pip
  - Set `YTDLP_PATH` and `NODE_PATH` env vars in Docker and `.env.example`
  - Update `backend/internal/services/ytdlp_service.go` to detect JS runtime availability and use explicit binary paths
  - Verify: test passes in CI and locally
- **Non-goals**: Multi-runtime JS support, yt-dlp auto-update mechanism

### Multi-environment YAML Config

- **Goal**: Support env-specific config overlays for different deployments.
- **Changes**:
  - Add `CONFIG_ENV` env var / `--config-env` CLI flag (values: `development`, `production`, `staging`)
  - Config loader reads `config.yaml` as base, then `config.<env>.yaml` deep-merge on top
  - Environment variable overrides remain highest priority (unchanged)
  - Ship `config.development.yaml` and `config.production.yaml`
  - Unit tests: merge logic, overlay precedence, missing file behavior
- **Non-goals**: Hot-reload, remote config sources, encrypted secrets

### LiteFS Worker Scaling

- **Goal**: Support multiple concurrent worker replicas sharing a SQLite database.
- **Changes**:
  - Add LiteFS sidecar (`flyio/litefs`) to `docker-compose.yml`
  - Configure FUSE mount at SQLite DB path with write-forwarding to a primary node
  - Allow `ops-worker` to scale (`replicas: 3` in compose or via `--scale`)
  - Validate `TableLockManager` behavior through LiteFS FUSE layer
  - Document scaling limits and LiteFS architecture in RUNBOOK.md
- **Non-goals**: Redis/NOTIFY-based job wakeup, Kubernetes support, multi-region replication

## Stream Œ≤: Testing

### CI Acceptance Test Fix

- **Goal**: Wire `acceptance_test.go` (integration-tagged) into CI without false negatives.
- **Changes**:
  - Add CI job for integration tests that starts required containers (Postgres, slskd)
  - Use `-tags=integration` for the acceptance test runner
  - Gate with `SKIP_INTEGRATION_TESTS` env var for non-integration runs
- **Non-goals**: Full integration suite in every PR (run on merge to master only)

### E2E Smoke Suite

- **Goal**: Comprehensive integration tests covering all core API paths.
- **Files** (`backend/internal/integration/`, all `//go:build integration`):
  - `auth_smoke_test.go` ‚Äî register, login, logout, rate-limit check
  - `watchlist_smoke_test.go` ‚Äî CRUD, sync trigger
  - `library_smoke_test.go` ‚Äî add, scan, list tracks
  - `job_smoke_test.go` ‚Äî create, poll, verify completion
  - `webhook_smoke_test.go` ‚Äî configure, trigger, verify delivery
  - `quota_smoke_test.go` ‚Äî exceed quota, verify warning
- Use existing `integration-tests.sh` runner pattern
- **Non-goals**: Performance/load tests, multi-worker race condition tests

### Mobile Nav & Keyboard Accessibility

- **Goal**: Verify and fix all pages at mobile viewport width and keyboard-only navigation.
- **Changes**:
  - Browser-audit at 375px viewport (or browser automation)
  - Fix missing focus indicators in CSS (visible focus rings)
  - Ensure sidebar collapse works at mobile breakpoints
  - Verify modal focus trap (Tab cycles within modal)
  - Test console and job pages keyboard-only
  - Add a11y expectations to CONTRIBUTING.md
- **Non-goals**: Screen reader certification, WCAG 2.2 AA full compliance audit

## Stream Œ≥: Admin Panel

### Backend

- `AdminHandler` in `backend/internal/api/admin_handler.go`
  - `GET /api/admin/users` ‚Äî list users with roles
  - `POST /api/admin/users` ‚Äî create user
  - `DELETE /api/admin/users/:id` ‚Äî delete user
  - `PATCH /api/admin/users/:id/role` ‚Äî toggle admin/user role
  - `POST /api/admin/users/:id/reset-password` ‚Äî force password reset
  - `GET /api/admin/audit` ‚Äî paginated activity log
  - `GET /api/admin/config` ‚Äî list Setting rows
  - `PATCH /api/admin/config` ‚Äî update Setting rows
- `AuditLog` model in database: `action`, `actor_id`, `target_type`, `target_id`, `timestamp`, `metadata` (JSON)
- Admin middleware check: `c.Locals("user").(database.User).Role == "admin"` on all routes
- Route registration in `backend/cmd/server/main.go` under `/api/admin/*`

### Frontend

- Admin layout page at `/admin` with sub-nav: Users | Audit Log | Config
- **Users**: Table with role badges, toggle button, delete confirmation modal
- **Audit Log**: Filterable, paginated table ‚Äî reuse existing table partial patterns
- **Config**: Key-value editor for Setting rows with validation and save button
- HTMX partials for admin operations (consistent with existing patterns)
- All admin routes protected by auth middleware + role check

### Audit Logging

- Service method `Audit(action, actorID, targetType, targetID, metadata)` called by handlers
- Metadata stored as JSON blob for extensibility
- Backfill existing actions? No ‚Äî start clean from v0.0.1

## Risk Register

| Risk | Likelihood | Mitigation |
|---|---|---|
| LiteFS FUSE forwarding breaks advisory locks | Medium | Test with `--scale ops-worker=3` before release; fallback: document as single-worker for v0.0.1 |
| yt-dlp test still fails after upgrade | Low | Pin yt-dlp version; test with known-working test URL |
| Admin panel audit log performance | Low | Paginate + index by timestamp; defer perf tuning until needed |
| Acceptance test CI job too slow | Medium | Gate to merge-to-master only; keep PR CI fast |
| Mobile audit finds layout regressions | Medium | Fix as found; no full redesign ‚Äî only fix breakage |
